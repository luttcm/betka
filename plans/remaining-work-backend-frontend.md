Источник: [plans/mvp-plan.md](plans/mvp-plan.md), [plans/frontend-iteration-plan.md](plans/frontend-iteration-plan.md), [AGENTS.md](AGENTS.md)

## Backend — что осталось

### P0 (критично для MVP)

- [x] Перевести хранение данных из in-memory в PostgreSQL для событий/ставок/кошелька (персистентность после рестарта).
- [x] Довести settlement-поток до полностью транзакционного сценария с корректными проводками кошелька (`hold/release/settle`) и audit.
- [x] Закрыть тестами e2e-сценарий MVP: `регистрация → создание события → модерация → ставка → settlement`.
- [x] Довести integration-тесты для транзакций БД и идемпотентности ставок.
- [x] Финализировать контракт API/ошибок и синхронизировать документацию в [docs/api.md](docs/api.md).

### P1 (важно после P0)

- [ ] Реализовать операции кошелька `subscribe/withdraw` (с валидациями и журналом транзакций).
- [ ] Работа кошелька будет в виде подписок на токены, дается первые 300 бесплатных, дальше надо покупать подписку, каждый день будет приходить по 100 токенов. Подписка ежемесячная, обычная и про, разница в том что будет приходить больше токенов. Токены вывести нельзя. Потом реализуем гача куртилку, чтоб за токены выбивать всякие штучки и обмениваться ими. + надо будет еще правильно распределить токены и реальные деньги, чтоб подписка была выгодная для нас, но и не душная для пользователя.
- [ ] Усилить observability: структурные логи, покрытие readiness/health, понятные сообщения ошибок.
- [ ] Проверить и зафиксировать индексы/ограничения в PostgreSQL под основные read/write сценарии.

### P2 (после стабилизации MVP)

- [ ] Расширить security-practices: rate limit, дополнительные RBAC проверки, аудит критичных действий.
- [ ] Подготовить нагрузочные/регрессионные проверки для odds/settlement.

## Frontend — что осталось

### P0 (критично для MVP)

- [x] Прогнать `lint/build` через Docker и устранить реальные регрессии (`docker compose run --rm --build frontend npm run lint` и `docker compose run --rm --build frontend npm run build`).
- [ ] Проверить весь e2e UX-флоу на реальном рантайме (auth/event/moderation/bet/wallet/settlement).
- [ ] Финально сверить UI с backend-контрактами после стабилизации PostgreSQL-репозиториев.

### P1 (важно после P0)

- [ ] Довести визуальную консистентность под выбранный light/VK-стиль (отступы, состояния кнопок, бейджи статусов).
- [ ] Улучшить UX форм: блокировки/сообщения/переходы в пограничных сценариях (401/403/409/422).
- [ ] Добавить визуальную проверку ключевых экранов (скриншоты/чеклист).

### P2 (после MVP)

- [ ] Пагинация и/или виртуализация длинных списков (события, транзакции, ставки).
- [ ] Улучшения accessibility (focus states, aria, контраст).
