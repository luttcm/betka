# Domain Rules

## События

- Пользователь создает событие
- Событие проходит модерацию
- Только `approved` события доступны для ставок

## Ставки

- Только виртуальная валюта
- При ставке баланс холдится
- Расчет при settlement события
- Коэффициенты динамические: чем больше объём ставок на исход, тем ниже коэффициент для новых ставок на этот исход
- Чем ниже оценочная вероятность исхода, тем выше потенциальный выигрыш
- Маржа платформы учитывается в котировке (через понижение справедливых odds)

## Модерация

- Роли `moderator` и `admin`
- Решение `approve/reject` протоколируется

## Завершение событий

- Создатель события инициирует завершение через запрос `request-settlement`
- В запросе обязательно передаётся доказательство: ссылка или файл (имя + base64-контент)
- После запроса событие переходит в статус `settlement_requested`
- Только `admin` выбирает итоговый исход (`yes|no`) и переводит событие в `settled`
- При финализации выполняется settlement ставок и выплаты победившим позициям
